# Security Policy

## Reporting a Vulnerability

This repository contains instructional content (Markdown files and code examples), not production software. However, if you find:

- **Code examples with security flaws** (e.g., SQL injection, command injection, hardcoded credentials)
- **Patterns that could lead to unsafe agent behavior** (e.g., missing input validation, privilege escalation)
- **Outdated security guidance** that no longer reflects best practices

Please report it by emailing **artur@thegeolab.net** or by opening a GitHub issue with the **Bug Report** template. Include:
- The specific file and section
- A description of the security concern
- A suggested fix if you have one

For content-only repositories like this one, public issue reporting is appropriate. There is no sensitive infrastructure to protect.

## Scope

This policy covers the content of this repository only: `SKILL.md`, `references/`, and any code examples within.

## Response Time

The maintainer reviews issues weekly and aims to address security-related reports within 7 days.
